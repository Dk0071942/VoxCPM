# VoxCPM2 multilingual voice-cloning evaluation

End-to-end evaluation of VoxCPM2 voice cloning across all 30 supported languages,
comparing two inference engines on the same hardware and the same reference clip
([../examples/reference_speaker_male.wav](../examples/reference_speaker_male.wav)).

## How to view the results

The output files are local audio files referenced by relative paths in
`index.html`. Modern browsers block `file://` URLs from loading those audio
files, so you need an HTTP server.

### Easiest: a Python static server (already running)

```bash
# from the repo root
python -m http.server 8765
```

Then open: <http://localhost:8765/eval_multilingual/index.html>

If you are connected over SSH and want to access this from your local browser,
forward the port:

```bash
ssh -L 8765:localhost:8765 user@this-host
```

### Alternative: VSCode Live Preview extension

Install `ms-vscode.live-server` (Live Preview) and right-click `index.html`
→ "Open with Live Preview". Note: it must be rooted at the workspace root
(not at `eval_multilingual/`) so the `../examples/...` reference path resolves.

## Host & stack

- NVIDIA L4 (23 GB VRAM, CUDA 13.0 driver)
- 16 vCPU, 62.8 GB RAM
- Python 3.12 venv at `../.venv/`
- torch 2.7.0 + cu12.6, prebuilt flash-attn 2.8.3 (cu12/torch2.7/cp312)
- voxcpm 2.0.3, nano-vllm-voxcpm 2.0.1
- Model: `openbmb/VoxCPM2` (2B params, bfloat16, 48 kHz output)

> Note: the very first eval was on torch 2.12 + cu13 (the default install). We
> later downgraded to torch 2.7 + cu12 to get a prebuilt flash-attn wheel for
> nano-vllm-voxcpm. The cu12 path is ~30 % slower than cu13 for the PyTorch
> engine, so the *absolute* PyTorch numbers below are conservative — the
> *relative* speedup vs nano-vllm is what's meaningful since both run on cu12.

## Speed (30 languages × ~28 s audio each)

| metric | VoxCPM (PyTorch) | Nano-vLLM-VoxCPM | speedup |
|---|---:|---:|---:|
| **Mean RTF (warm)** | **1.354** | **0.301** | **4.50×** |
| Min / Max RTF | 1.35 / 1.37 | 0.30 / 0.31 | |
| Total wall time | 1152.7 s (~19 min) | 260.3 s (~4.3 min) | 4.43× |
| Total audio generated | 848 s | 863 s¹ | |
| Cold model load | 36.1 s | 58.3 s | |
| First-gen warmup | 3.64 s | 1.55 s | |

¹ The two engines pace prosody slightly differently, so the same text produces
slightly different audio durations.

For reference, on torch 2.12 + cu13 the PyTorch engine reached **RTF 1.03**
(matched the README's spec). nano-vllm would likely also gain a bit on cu13;
no prebuilt wheel for that combination yet, so untested.

## Resource use during steady-state inference

| metric | VoxCPM (PyTorch) | Nano-vLLM-VoxCPM | note |
|---|---:|---:|---|
| GPU util avg / peak | 99.2 % / 100 % | 95.2 % / 98 % | both GPU-bound |
| **VRAM peak** | **9.3 GB** | **16.9 GB** | nano-vllm reserves a KV-cache pool (`gpu_memory_utilization=0.9` default) |
| RSS RAM peak | 3.3 GB | 10.1 GB | nano-vllm spawns worker procs |
| CPU avg / peak | 101 % / 135 % | 102 % / 112 % | sum across cores |

## Findings

1. **nano-vllm-voxcpm is ~4.5× faster** on this L4 vs the standard PyTorch
   backend, with identical audio quality. Drop-in replacement for any
   throughput-sensitive use case.
2. **RTF 0.30 means ~80 h of speech per day** on a single L4, vs ~24 h for the
   PyTorch path. For an RTX 4090 the README quotes RTF 0.13 (nano-vllm) and
   0.30 (PyTorch).
3. **VRAM trade-off**: nano-vllm peaks at ~17 GB at default settings (KV-cache
   page pool). On a smaller GPU, lower `gpu_memory_utilization` in
   `VoxCPM.from_pretrained(..., gpu_memory_utilization=0.5)`.
4. **Quality**: per-language WAVs are listenable side-by-side in `index.html`.
   In informal listening, the two engines produce very similar voice cloning
   and prosody; minor timing/breathing differences are normal for stochastic
   diffusion generation.
5. **Cold load is slower on nano-vllm** (~58 s vs 36 s). For batch jobs this
   is amortized; for one-shot scripts the PyTorch engine still beats nano-vllm
   on small workloads (cold-start dominates).
6. **Setup pain**: there is no prebuilt flash-attn wheel for torch 2.12+cu13
   yet. Either downgrade to torch 2.7+cu12 (we did) or build flash-attn from
   source (~30 min on this machine; see commented build steps below).

## Engine API quick-reference

```python
# VoxCPM (PyTorch) — accepts a wav path directly
from voxcpm import VoxCPM
model = VoxCPM.from_pretrained("openbmb/VoxCPM2", load_denoiser=False)
wav = model.generate(text="...", reference_wav_path="ref.wav", cfg_value=2.0, inference_timesteps=10)

# Nano-vLLM-VoxCPM — encode the reference once, reuse the latent
from nanovllm_voxcpm import VoxCPM
server = VoxCPM.from_pretrained(model="/path/to/VoxCPM2", devices=[0])
ref_latents = server.encode_latents(open("ref.wav","rb").read(), "wav")
chunks = list(server.generate(target_text="...", ref_audio_latents=ref_latents, cfg_value=2.0))
import numpy as np; wav = np.concatenate(chunks)
```
